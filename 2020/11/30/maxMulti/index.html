题目：乘积最大

给定N个整数A1, A2, … AN。请你从中选出K个数，使其乘积最大。
请你求出最大的乘积，由于乘积可能超出整型范围，你只需输出乘积除以1000000009的余数。
注意，如果X<0， 我们定义X除以1000000009的余数是负(-X)除以1000000009的余数。
即：0-((0-x) % 1000000009)
【输入格式】
第一行包含两个整数N和K。
以下N行每行一个整数Ai。

对于40%的数据，1 <= K <= N <= 100
对于60%的数据，1 <= K <= 1000
对于100%的数据，1 <= K <= N <= 100000 -100000 <= Ai <= 100000

【输出格式】
一个整数，表示答案。
【输入样例】
5 3
-100000
-10000
2
100000
10000
【输出样例】
999100009

再例如：
【输入样例】
5 3
-100000
-100000
-2
-100000
-100000
【输出样例】
-999999829

题目分析

注意：这题是乘积最大，不是最后的模最大！
暴力搜索对于这一题来说肯定是不行的，首先会超时，其次最大乘积可能无法记录(有可能超过long long)
那么我们来换一种思路
像一道比较类似的题目：和最大。 求最大和可以用动态规划，也可以只取正数，很快能得到结果
但是乘法呢，两个负数相乘就变成了正数，因此问题就变得比较复杂，需要分情况讨论
这里先将这些数排序
先考虑比较简单的情况，全是正数，那么只要取最大的k个就好了
其次，全是负数的情况。这里需要分成两种情况讨论：1.k为偶数 2.k为奇数
  k为偶数，那么最后乘积为正数，因此取绝对值最大的k个负数，也就是最小的k个数，即前k个
  k为奇数，那么最后乘积为负数，因此要选绝对值最小的k个数，也就是排序后的最后k个数
最后，也是比较复杂的，既有正数又有负数的情况
先举个例子:-20,-19,-18,0,1,2,3,6,9 n = 9
当k = n = 9时，乘积只能是所有数相乘
而当k < n时，应为有正有负，总能使乘积为正(负数取偶数个)**，因此转化为枚举取哪几个负数，哪几个正数**
因此，枚举负数的个数，首先个数为0时，就是取最后k个数(有可能乘积为负，没关系，等之后调整)
取2个负数(绝对值大的)**，即前两个数的时候，计算他们的乘积；在求出正数中最小的2个数的乘积，比较他们的大小。若负数的乘积大，更新答案，否则退出循环枚举，前一次的乘积即为结果
之后的同理，每次比较负数的两个数乘积与正数两个乘积的大小**即可
